# mynewgame

Тайловая 2D-игра на основе карты, сделанной в [Tiled](https://www.mapeditor.org/).

## Что здесь лежит

| Папка | Что внутри |
|---|---|
| `Tiled_files/` | Карта `Forest` и тайлсеты, которые она использует. Это то, что нужно игре. |
| `PNG/` | Экспортированные тайлсеты и отдельные объекты (`Objects_separated/`, 89 спрайтов). |
| `PSD/` | Исходники графики для Photoshop. Игре не нужны, нужны для правок арта. |

## Карта

`Tiled_files/Forest.tmx` открывается в Tiled (версия 1.12.2 или новее). Параметры:

- размер тайла 16×16
- бесконечная карта (данные хранятся чанками)
- ортогональная проекция
- вода анимирована через кадры в тайлсете `water_detilazation.png`

`Forest.tmj` — та же карта в JSON. Многим движкам удобнее читать JSON, а Tiled по-прежнему открывает `.tmx`. Если правишь карту, экспортируй `.tmj` заново, иначе форматы разъедутся.

Тайлсеты лежат рядом с картой в `Tiled_files/`, и пути в карте относительные — не перемещай PNG-файлы из этой папки, иначе Tiled потеряет тайлсеты.

## Как начать работу

```sh
git clone https://github.com/dak9n/mynewgame.git
cd mynewgame
```

Дальше открой `Tiled_files/Forest.tmx` в Tiled.

## Совместная разработка

Карты Tiled — это XML/JSON, git умеет их сливать, но конфликт в `.tmx` руками разбирать больно. Поэтому:

- договоритесь, кто правит карту в конкретный момент, либо работайте в разных слоях/областях;
- перед началом работы делайте `git pull`, после — сразу коммитьте и пушьте;
- PSD слить невозможно в принципе: git при конфликте оставит один из файлов целиком. Не редактируйте один и тот же PSD одновременно.
