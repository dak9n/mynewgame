# mynewgame

Тайловая 2D-игра про лес. TypeScript + [Phaser 3](https://phaser.io/), сборка через Vite.

## Запуск

```sh
npm install
npm run dev
```

Откроется на http://localhost:5173. Карту можно таскать мышью и зумить колесом.

| Команда | Что делает |
|---|---|
| `npm run dev` | Дев-сервер с горячей перезагрузкой |
| `npm run build` | Проверка типов + сборка в `dist/` |
| `npm run convert-map` | Разовая миграция карты из Tiled (см. ниже) |

## Карта

**Источник правды — `public/assets/maps/forest.json`.** Это наш собственный формат, его читает игра и будет править редактор. Описание полей — в [src/map/types.ts](src/map/types.ts).

Карта — 48×32 тайла по 16 пикселей (768×512 px), 26 слоёв, 10 тайлсетов, 624 анимированных тайла (вода и кувшинки).

Устройство формата:

- слои — плоские массивы `width * height`, а не чанки: `0` — пусто, иначе глобальный номер тайла;
- глобальный номер = `firstId` тайлсета + номер тайла внутри него;
- в трёх старших битах номера — флаги отражения и поворота (кодировка Tiled). Разбирать их вручную не надо, для этого есть `Phaser.Tilemaps.Parsers.Tiled.ParseGID`;
- тайлсеты встроены в json целиком, включая анимации — внешних файлов формат не требует.

## Откуда взялась карта и при чём тут Tiled

Изначально карта рисовалась в Tiled и лежит в `Tiled_files/` (`Forest.tmx` / `Forest.tmj`). Скрипт [tools/convert-map.mjs](tools/convert-map.mjs) один раз перевёл её в наш формат: развернул чанки, встроил внешний тайлсет `Water_detilazation2.tsx` и обрезал карту по реально нарисованному.

**Больше запускать его не нужно.** Он перезапишет `forest.json` и сотрёт всё, что нарисовано в редакторе, — поэтому он отказывается работать, если файл уже существует (обойти можно флагом `--force`, но подумай дважды).

`Tiled_files/` и `PSD/` остаются как архив исходников. Игра их не читает.

## Структура

| Путь | Что это |
|---|---|
| `src/main.ts` | Точка входа, конфиг Phaser |
| `src/scenes/WorldScene.ts` | Рендер карты, анимация тайлов, камера |
| `src/map/types.ts` | Формат карты |
| `tools/convert-map.mjs` | Разовая миграция из Tiled |
| `public/assets/` | Карта и картинки тайлсетов — отдаются как есть |
| `Tiled_files/`, `PSD/` | Архив исходников |

## Совместная разработка

`forest.json` — один файл на всю карту, и git не сможет осмысленно слить правки двоих. Если вы оба поправите карту, кто-то потеряет работу. Пока редактора нет, договаривайтесь, кто её трогает; когда появится — стоит подумать про формат, который сливается лучше (например, слои отдельными файлами).

PSD не сливаются вообще — там то же правило, только жёстче.
